# 应用架构文档

## 1. 项目概述

### 1.1 项目背景
「南网在线」APP 是中国南方电网有限责任公司官方推出的智能用电服务平台，为南方五省（广东、广西、云南、贵州、海南）用电客户提供全方位的电力服务。本文档描述该应用的整体架构设计。

### 1.2 设计目标
- **高可用**：核心服务可用性 ≥ 99.95%
- **高性能**：接口响应 P99 ≤ 500ms
- **高安全**：满足等保三级要求
- **易扩展**：支持亿级用户、千万级日活

## 2. 整体架构

### 2.1 架构总览

```
┌─────────────────────────────────────────────────┐
│                    客户端层                       │
│   iOS APP  │  Android APP  │  H5  │  小程序     │
├─────────────────────────────────────────────────┤
│                    接入层                         │
│      CDN  →  WAF  →  API Gateway  →  BFF       │
├─────────────────────────────────────────────────┤
│                   业务服务层                      │
│  用户中心 │ 电费服务 │ 工单服务 │ 支付服务 │ ...  │
├─────────────────────────────────────────────────┤
│                   中间件层                        │
│  消息队列 │ 分布式缓存 │ 配置中心 │ 服务注册      │
├─────────────────────────────────────────────────┤
│                    数据层                        │
│  MySQL集群 │ Redis集群 │ Elasticsearch │ OSS    │
├─────────────────────────────────────────────────┤
│                   基础设施层                      │
│  K8s集群 │ 日志平台 │ 监控告警 │ CI/CD           │
└─────────────────────────────────────────────────┘
```

### 2.2 架构原则
1. **微服务架构**：按业务域拆分服务，独立部署、独立扩展
2. **前后端分离**：客户端通过 BFF 层聚合接口
3. **事件驱动**：关键业务事件通过消息队列异步处理
4. **数据一致**：核心业务采用分布式事务保障一致性
5. **安全纵深**：多层安全防护，端到端加密

## 3. 客户端架构

### 3.1 技术选型

| 平台 | 技术方案 | 说明 |
|------|----------|------|
| iOS | Swift + SwiftUI | 原生开发，最佳性能 |
| Android | Kotlin + Jetpack Compose | 原生开发，Material Design 3 |
| H5 | React + TypeScript | Web端支持 |
| 小程序 | 微信小程序原生 | 微信生态覆盖 |

### 3.2 客户端分层

```
┌─────────────────────────────┐
│        展现层 (UI Layer)     │
│   页面、组件、动画、主题      │
├─────────────────────────────┤
│      业务层 (Feature Layer)  │
│   业务逻辑、状态管理          │
├─────────────────────────────┤
│      基础层 (Foundation)     │
│   网络、存储、日志、埋点      │
├─────────────────────────────┤
│      能力层 (Capabilities)   │
│   推送、定位、摄像头、生物识别 │
└─────────────────────────────┘
```

### 3.3 关键技术方案

- **网络层**：OkHttp / URLSession，支持 HTTP/2、证书锁定
- **图片加载**：Glide / Kingfisher，三级缓存（内存→磁盘→网络）
- **数据持久化**：SQLite + MMKV / CoreData，敏感数据 Keychain
- **状态管理**：Redux / MVVM，单向数据流
- **埋点分析**：自研埋点 SDK，支持全埋点 + 代码埋点
- **热修复**：Android Tinker / iOS JSPatch（仅限紧急修复）

## 4. 服务端架构

### 4.1 微服务划分

| 服务 | 职责 | 技术栈 |
|------|------|--------|
| 用户中心 | 注册登录、实名认证、用户信息 | Java/Spring Boot |
| 账户服务 | 户号管理、多户关联 | Java/Spring Boot |
| 电费服务 | 电费查询、账单生成、用电分析 | Java/Spring Boot |
| 工单服务 | 报装、报修、变更工单流转 | Java/Spring Boot |
| 支付服务 | 缴费、退款、对账 | Java/Spring Boot |
| 通知服务 | 短信、推送、站内信 | Go |
| 内容服务 | 资讯、公告、视频 | Python/FastAPI |
| 分析服务 | 用电分析、数据报表 | Python/Spark |
| 客服服务 | 智能客服、工单流转 | Python |

### 4.2 服务通信

```
同步调用：gRPC (服务间) + HTTP/REST (BFF→客户端)
异步通信：Apache Kafka (事件总线)
服务发现：Nacos
配置管理：Nacos Config
负载均衡：Nginx + Spring Cloud Gateway
熔断降级：Sentinel
```

### 4.3 API 网关设计

```
客户端请求
    ↓
[CDN] → 静态资源加速
    ↓
[WAF] → SQL注入/XSS/CC攻击防护
    ↓
[API Gateway]
    ├── 路由转发
    ├── 鉴权 (JWT + OAuth2)
    ├── 限流 (令牌桶)
    ├── 熔断 (Sentinel)
    ├── 日志 (请求追踪)
    └── 协议转换 (HTTP ↔ gRPC)
    ↓
[BFF层]
    ├── Mobile BFF (移动端聚合)
    └── Web BFF (Web端聚合)
    ↓
[微服务集群]
```

## 5. 数据架构

### 5.1 数据库设计

| 存储 | 用途 | 方案 |
|------|------|------|
| 业务主库 | 用户、订单、工单 | MySQL 8.0 主从集群 |
| 缓存 | 会话、热点数据、验证码 | Redis Cluster |
| 搜索 | 停电信息、营业网点 | Elasticsearch |
| 文件 | 发票、合同、照片 | 阿里云 OSS / MinIO |
| 时序数据 | 电表数据、用电量 | InfluxDB / TDengine |

### 5.2 数据流

```
智能电表 → 数据采集 → Kafka → 实时计算(Flink) → 时序数据库
                                              ↓
                                        用电分析服务
                                              ↓
                                        用户用电报告

用户缴费 → 支付服务 → 对账系统 → 电费系统 → 电网调度系统
              ↓
         通知服务 → 短信/推送
```

### 5.3 数据安全

- **传输加密**：全链路 TLS 1.3
- **存储加密**：敏感数据 AES-256 加密存储
- **脱敏展示**：身份证号、手机号、银行卡号展示脱敏
- **数据分类**：按敏感等级分级管理
- **审计日志**：全量操作审计，保留 180 天

## 6. 安全架构

### 6.1 安全体系

```
┌─────────────────────────────────────┐
│          应用安全                     │
│  防篡改 │ 代码混淆 │ 完整性校验       │
├─────────────────────────────────────┤
│          通信安全                     │
│  TLS 1.3 │ 证书锁定 │ 双向认证       │
├─────────────────────────────────────┤
│          身份安全                     │
│  生物识别 │ 设备指纹 │ 多因素认证     │
├─────────────────────────────────────┤
│          数据安全                     │
│  加密存储 │ 脱敏展示 │ 访问控制       │
├─────────────────────────────────────┤
│          基础安全                     │
│  WAF │ DDoS防护 │ 入侵检测 │ HIDS   │
└─────────────────────────────────────┘
```

### 6.2 认证授权

- **用户认证**：手机号 + 短信验证码（主），密码（辅）
- **生物识别**：指纹/FaceID 快速登录
- **实名认证**：OCR 身份证 + 联网核查 + 人脸识别
- **授权协议**：OAuth 2.0 + JWT Token（有效期 2h，Refresh Token 7d）
- **会话管理**：Redis 存储会话状态，支持单设备/多设备策略

### 6.3 合规要求

- 等保三级认证
- 个人信息保护法（PIPL）合规
- 网络安全法合规
- 南方电网内部安全标准

## 7. 部署架构

### 7.1 部署拓扑

```
                    ┌──── 公有云 (阿里云/华为云) ────┐
                    │                                │
用户 ──→ CDN ──→ SLB ──→ K8s集群                     │
                    │   ├── Web服务 Pod × N          │
                    │   ├── API服务 Pod × N          │
                    │   ├── 任务调度 Pod × N         │
                    │   └── 中间件 Pod × N           │
                    │                                │
                    │   RDS(MySQL主从)               │
                    │   Redis集群                    │
                    │   Elasticsearch集群            │
                    │   Kafka集群                    │
                    │   OSS对象存储                  │
                    │                                │
                    └────────────────────────────────┘
                              │
                              │ VPN专线
                              ↓
                    ┌──── 电网内网 ────┐
                    │  电网调度系统     │
                    │  智能电表系统     │
                    │  营销系统        │
                    └─────────────────┘
```

### 7.2 容量规划

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 注册用户 | 1亿+ | 南方五省覆盖 |
| 日活用户 | 1000万 | 高峰时段集中 |
| 并发请求 | 10万 QPS | 缴费高峰 |
| 数据存储 | 50TB+ | 含电表数据 |
| 文件存储 | 100TB+ | 发票、照片 |

### 7.3 高可用策略

- **多可用区部署**：至少 2 个可用区，故障自动切换
- **数据库主从**：MySQL 一主两从，半同步复制
- **缓存高可用**：Redis Cluster 6 节点，自动故障转移
- **弹性伸缩**：K8s HPA，根据 CPU/QPS 自动扩缩容
- **异地容灾**：核心数据异地备份，RPO < 1min，RTO < 5min

## 8. 监控运维

### 8.1 监控体系

| 监控维度 | 工具 | 指标 |
|----------|------|------|
| 基础设施 | Prometheus + Grafana | CPU、内存、磁盘、网络 |
| 应用性能 | SkyWalking | 接口耗时、错误率、吞吐量 |
| 业务监控 | 自研大盘 | 缴费成功率、工单完成率 |
| 日志分析 | ELK Stack | 错误日志、安全审计 |
| 告警 | AlertManager | P0: 电话+短信, P1: 钉钉, P2: 邮件 |

### 8.2 CI/CD 流程

```
代码提交 → GitLab CI
    ↓
代码扫描 (SonarQube)
    ↓
单元测试 (JUnit/Jest)
    ↓
构建镜像 (Docker)
    ↓
集成测试 (Testcontainers)
    ↓
灰度发布 (Canary)
    ↓
全量发布 (Blue-Green)
    ↓
监控告警
```
